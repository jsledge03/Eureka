1) Clarify “Strategic Intent” (definition + correct content)

1A) Define Strategic Intent (in-app copy + tooltip)

Strategic Intent should answer:
	•	“What am I optimizing for right now?”
	•	“What am I protecting?”
	•	“What tradeoffs am I willing to make this season/quarter?”

It should be a short, human-readable statement that guides decisions and prioritization.

Add a ⓘ tooltip explaining:
	•	Strategic Intent is not metrics or settings.
	•	It’s a leadership stance: outcomes + constraints + values.

1B) Remove unrelated items from Strategic Intent UI

Strategic Intent should NOT show:
	•	Grace Bank
	•	Daily Rhythm
	•	Set Theme

Those can exist elsewhere, but remove them from the Strategic Intent section so it stays conceptually clean.

If any of those are currently embedded there, relocate them:
	•	Grace Bank → Coach & Insights (Accountability tools)
	•	Daily Rhythm → My Day / Coach
	•	Theme → Quarter plan header or Strategic Trajectory (if appropriate)

2) Clarify “Strategic Trajectory” (definition + best use)

2A) Define Strategic Trajectory (in-app copy + tooltip)

Strategic Trajectory should answer:
	•	“Where is my system heading if I keep operating this way?”
	•	“What’s my next evolution in identity, domains, and goals?”
	•	“What is the next 2–6 week directional move?”

It is trend + direction, not a plan.
It should synthesize:
	•	momentum/rhythm trends
	•	domain drift vs emphasis
	•	friction patterns
	•	forecast outlook
	•	recent proof (identity evidence)

Add a ⓘ tooltip:
	•	Strategic Trajectory is a directional readout + recommended recalibration.
	•	It helps you steer the operating system.

2B) Enhance the field: make it meaningful and actionable

Strategic Trajectory should show:
	1.	Direction summary (1–2 sentences)
	2.	“Tailwinds” (what’s working)
	3.	“Headwinds” (what’s costing you)
	4.	“3 recommended course corrections” with buttons that DO things:
	•	reduce scope / pause habit
	•	schedule reset
	•	shift to Minimum Version defaults
	•	adjust constraints / not-to-do
	•	prioritize a keystone habit

All actions must route correctly and persist.
