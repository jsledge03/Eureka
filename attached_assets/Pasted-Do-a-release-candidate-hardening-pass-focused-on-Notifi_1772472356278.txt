D) TUNE DRIFT THRESHOLDS (make it smart, not noisy)

Tune thresholds so drift is meaningful but not spammy, using these defaults:

D1) Overall drift severity
	•	LOW: 1–2 mild signals
	•	MEDIUM: 3+ mild signals OR 1 strong signal
	•	HIGH: 2+ strong signals OR overdue spiral + habit misses + low reflection

D2) Specific tuned thresholds
	1.	Habit commitment drift:

	•	Flag only if a habit is missed >= 3 times in 7 days AND user has NOT adapted (no Minimum Version used, no reschedule, no pause)

	2.	Domain drift:

	•	Flag only if:
	•	domain emphasis weight >= 2 AND
	•	7 days with near-zero aligned actions in that domain
(avoid flagging if user is in Recover mode and explicitly deprioritized domain)

	3.	Reflection drift:

	•	Flag if Close-the-loop < 2 days/week AND weekly vibe check skipped

	4.	Overdue drift:

	•	Flag if overdue count increased week-over-week AND > 3 overdue tasks

	5.	Overload drift:

	•	Flag if user logged “Overloaded/Behind” >= 3 times in a week and did not triage

D3) Reduce alert noise
	•	Show max 3 drift alerts at once (highest impact)
	•	Add “Snooze drift alerts for 7 days” option
