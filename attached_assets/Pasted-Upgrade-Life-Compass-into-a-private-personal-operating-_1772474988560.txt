1A) What it is

Season Mode represents the user’s current life context:
	•	Build (default)
	•	Stabilize
	•	Recover
	•	Sprint
	•	Reposition

1B) Data model

Add:
	•	seasonMode: enum
	•	seasonStartDate
	•	seasonNotes (optional)
	•	seasonDefaults object (thresholds + expectations)

1C) UI integration

Add Season Mode selector:
	•	Location: Coach & Insights at top near North Star OR My Day header
	•	UX: a pill (“Season: Build ▾”) opens a bottom sheet with:
	•	mode descriptions
	•	what changes (expectations)
	•	“Set Season” button

1D) How Season changes behavior (must be real)

Season Mode must modify:
	•	default grace days/week
	•	drift threshold sensitivity
	•	notification frequency (gentle)
	•	daily rhythm expectations (minimum commitments)
	•	recommended task load for Today

Example defaults (tunable later):
	•	Build: normal drift sensitivity, grace=2, task target=5
	•	Sprint: higher task target, lower grace (but still available), higher drift sensitivity for overload
	•	Recover: low task target, higher grace, drift focuses on recovery + reflection rather than productivity

These defaults must be visible and editable in advanced settings.

3A) Purpose

Create a timeline of “proof” moments supporting the active identity statement.

A ProofEvent is created when:

	•	user completes a linked goal task
	•	user adapts wisely on low energy (uses Minimum Version + completes 1 aligned action)

ProofEvent structure:

3C) UI integration

4) Keystone Gravity Mapping (What Actually Moves the System)

4A) Definition

Identify which habits most strongly influence positive outcomes:
	•	higher rhythm
	•	higher identity alignment
	•	fewer overdue tasks
	•	better energy trend (optional)

4B) Computation (local + simple, no fake precision)

For each habit:
Compute a “Gravity Score” based on correlation-like heuristics over last 28 days:
	•	Compare days habit completed vs not completed:
	•	avg rhythm next day
	•	avg overdue count change
	•	avg identity alignment
Keep it simple and robust; avoid heavy stats.

GravityScore output:
	•	0–100
	•	label: High leverage / Medium / Low

4C) UI integration

In Habits Hub:
	•	Add “Keystone Gravity” indicator next to habits (subtle icon + label)
	•	Add a “High Leverage” filter
In Coach:
	•	Suggest focusing on 1–2 high leverage habits during overload or reset
