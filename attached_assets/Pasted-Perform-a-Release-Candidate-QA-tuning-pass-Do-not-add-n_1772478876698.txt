3) Drift thresholds tuning (reduce noise; increase signal)

Tune drift engine thresholds to be meaningful and calm:

Defaults:
	•	Habit drift: missed ≥3 in 7 days AND no adaptation (no minimum version/reschedule/pause)
	•	Domain drift: emphasized domain (weight ≥2) has near-zero aligned actions for 7 days (skip flagging if Season=Recover and domain is deprioritized)
	•	Overdue drift: overdue count >3 AND week-over-week increase
	•	Reflection drift: close-the-loop <2/week AND weekly vibe check skipped
	•	Overload drift: “Behind/Overloaded” logged ≥3/week AND no triage action applied

Rules:
	•	Max 3 drift alerts displayed at once
	•	Add snooze option (7 days)
	•	Drift must recommend only actionable fixes with working buttons

Verify drift alerts appear in Identity + Coach and actions apply real changes.

4) Forecast tuning (Adaptive Capacity Forecasting)

Tune forecast thresholds so it’s predictive but not alarmist:

Signals:

	•	drift severity
	•	season mode

Outputs:
	•	Stable / Watch / Overload Risk / Recovery Needed

Rules:

	•	Provide 2–3 actions; all must work (reduce scope, focus mode, minimum version, schedule reset)
