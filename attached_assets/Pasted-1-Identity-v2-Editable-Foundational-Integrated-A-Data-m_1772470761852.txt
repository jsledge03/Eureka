3) Refine Identity Alignment Algorithm (0–100) + Tooltips

Implement a transparent rubric that is stable, intuitive, and non-shaming.

A) Compute daily Identity Alignment (0–100)

Daily components:
	1.	Intentionality (0–30)

	•	+10 if user set Top outcomes for the day
	•	+10 if user selected focus domains (up to 3)
	•	+10 if user completed at least 1 keystone habit OR 1 “Most Important Task”

	2.	Congruent Action (0–40)
Compute proportion of completed actions that are aligned:
Aligned actions = completed tasks/habits linked to:

	•	a goal OR
	•	a focus domain OR
	•	a label that maps to a domain emphasis OR
	•	a value/commitment tag (optional)
Score = min(40, 40 * alignedCompleted / max(1, totalCompleted))

	3.	Integrity Under Constraints (0–20)
If energy ≤ 2 or mode=Recover:

	•	+20 if user used Minimum Version or reduced scope (triage) AND still completed 1 aligned action
	•	+10 if user used Minimum Version OR triage
	•	0 if user forced full load and completed nothing (but do not punish further)
If energy ≥ 3:
	•	+20 if user maintained commitments with sustainable completion (not overload)

	4.	Reflection (0–10)

	•	+10 if user completed Close the Loop
	•	+5 if user wrote at least 1 reflection prompt response
	•	+0 if skipped (no negative points beyond absence)

Daily Identity Alignment = sum of components.

B) Weekly Identity Alignment (0–100)

Weekly score = weighted average:
	•	60% average daily alignment
	•	40% “evidence stability” (did user have ≥3 days with alignment >60?)

Add trend:
	•	compare current week vs last week.

C) Tooltip copy

Use the microcopy provided earlier; include weights and the “compass not grade” line.

QA:
	•	Score should not crater on low activity days
	•	Score should respond to reflection and triage actions
	•	Tooltips visible and accurate

4) Identity Drift Detection Engine (advanced, non-shaming)

The engine detects sustained patterns that contradict identity commitments, then suggests system redesign actions.

A) Drift signals (compute weekly)

Compute drift risk per domain and overall (Low / Medium / High).

Signals:
	1.	Domain Drift

	•	if a domain emphasized in identity (weight ≥2) is neglected:
	•	(habit completion in that domain below threshold) OR
	•	(no tasks completed in that domain for ≥7 days)

	2.	Reflection Drift

	•	close-the-loop completion below threshold (e.g., <2/week)

	3.	Integrity Drift

	•	repeated “overload” midday check-ins without scope reduction

	4.	Commitment Drift

	•	same habit missed ≥3 times in 7 days and not adapted (no Minimum Version use, no schedule change)

	5.	Overdue Drift

	•	overdue count increasing week-over-week

B) Drift interpretation layer (human language)

For each drift flag, generate:
	•	“What’s happening”
	•	“Why it matters to your identity”
	•	“Likely cause” (capacity, friction, cue mismatch, energy mode, too many commitments)

C) Drift actions (must be clickable)

For each drift flag, provide 1–3 actions:
	•	“Simplify this habit (Minimum Version)”
	•	“Reschedule to a realistic time”
	•	“Pause this goal for this quarter”
	•	“Reduce active goals to 3”
	•	“Create a 15-min overdue cleanup sprint”
	•	“Add a Sunday Reset reminder”
	•	“Add a boundary constraint”

Actions must route to the correct editor and apply changes.

D) Drift UI

Add a “Drift” sheet accessible from:
	•	Identity page (“View Drift”)
	•	Coach & Insights (“Drift alerts”)

Display:
	•	Overall drift status (Low/Med/High)
	•	Top 3 drift alerts (cards)
	•	One-tap fixes

E) Guardrails (avoid shaming)
	•	Use calm language: “Your system is miscalibrated,” not “you failed.”
	•	Drift is always framed as redesign opportunity.
	•	Provide gentle, limited alerts (max 3 at once).

QA:
	•	Drift sheet loads even with empty data
	•	Actions apply changes
	•	No dead buttons
	•	Reload persistence
